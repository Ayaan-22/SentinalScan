Migrate crawler to httpx + asyncio
Replace requests.Session + ThreadPoolExecutor with AsyncClient and asyncio.gather()
CRITICAL Performance
8 hrs critical
What & why
The current crawler uses ThreadPoolExecutor with a manual future-tracking loop that has a logic bug — as_completed() is re-created each iteration so futures never correctly drain. Migrating to httpx.AsyncClient with a semaphore-gated asyncio queue fixes the bug and cuts scan time by ~60% on I/O-bound targets. Execute in 4 sub-steps to avoid a big-bang rewrite.
Implementation

# Sub-step 2: convert crawl_page() to async, run via asyncio.run()
# Sub-step 3: convert tester methods to async
# Sub-step 4: full async pipeline, remove ThreadPoolExecutor

# backend/crawler_async.py (sub-step 4)
import httpx, asyncio
from selectolax.parser import HTMLParser

class AsyncWebCrawler:
    async def __aenter__(self):
        self.client = httpx.AsyncClient(
            headers={"User-Agent": self.config.user_agent},
            verify=self.config.verify_ssl,
            timeout=self.config.timeout,
            limits=httpx.Limits(max_connections=self.config.workers)
        )
        return self

    async def crawl(self) -> list[str]:
        queue = asyncio.Queue()
        await queue.put(self.base_url)
        visited, pages = set(), []
        sem = asyncio.Semaphore(self.config.workers)

        async def worker():
            while True:
                url = await queue.get()
                if url in visited or len(visited) >= self.config.max_pages:
                    queue.task_done(); continue
                visited.add(url)
                async with sem:
                    resp = await self._fetch(url)
                if resp:
                    pages.append(url)
                    for lnk in self._extract_links(url, resp.text):
                        if lnk not in visited:
                            await queue.put(lnk)
                queue.task_done()

        workers = [asyncio.create_task(worker())
                   for _ in range(self.config.workers)]
        await queue.join()
        for w in workers: w.cancel()
        return pages

Parallelize vulnerability tests per page
asyncio.gather() collapses per-page test time from sequential to worst-single-test
HIGH Performance
5 hrs high
What & why
test_sqli() sleeps 3s per time-based payload. With 10 forms and 3 payloads each, one page can take 90 seconds sequentially. Running all tests for a page concurrently via asyncio.gather() collapses this to the slowest single test (~3s), a 30× improvement for pages with many forms.
Implementation
async def test_page(self, url: str, response: httpx.Response):
    forms = self._extract_forms(url, response.text)

    page_tasks = [
        self.test_open_redirect(url),
        self.test_path_traversal(url),
        self.test_info_disclosure(url, response.text),
        self.test_security_headers(url, response),
        self.test_cookie_security(url, response),
        self.test_sensitive_files(url),
        *[self._test_form(f, url) for f in forms],
    ]
    results = await asyncio.gather(*page_tasks, return_exceptions=True)

    vulns = []
    for r in results:
        if isinstance(r, list): vulns.extend(r)
        elif isinstance(r, Vulnerability): vulns.append(r)
        elif isinstance(r, Exception):
            self.logger.warning(f"Test error: {r}")
    return vulns

async def _test_form(self, form, url: str) -> list[Vulnerability]:
    results = await asyncio.gather(
        self.test_xss(form, url),
        self.test_sqli(form, url),
        self.test_csrf(form, url),
        return_exceptions=True
    )
    return [r for r in results if isinstance(r, Vulnerability)]

import redis.asyncio as redis, json, os

_r = redis.from_url(os.environ["REDIS_URL"])

async def save_scan_status(scan_id: str, status: dict, ttl: int = 86400):
    await _r.setex(f"scan:{scan_id}:status", ttl, json.dumps(status))

async def append_result(scan_id: str, vuln: dict):
    await _r.rpush(f"scan:{scan_id}:results", json.dumps(vuln))
    await _r.expire(f"scan:{scan_id}:results", 86400)

async def get_results(scan_id: str) -> list[dict]:
    raw = await _r.lrange(f"scan:{scan_id}:results", 0, -1)
    return [json.loads(r) for r in raw]

    await _r.publish(f"logs:{scan_id}", json.dumps(entry))

Decompose VulnerabilityScanner.scan()
Split the 150-line method into _phase_crawl(), _phase_test(), _summarize()
MEDIUM Refactor
3 hrs medium
What & why
The 150-line scan() method does authorization, URL validation, crawl orchestration, vulnerability testing, deduplication, and console printing. Each responsibility should be in its own method so they can be tested, overridden, and reasoned about independently.
Implementation
class VulnerabilityScanner:

    async def scan(self) -> list[Vulnerability]:
        self._verify_authorization()   # raises PermissionError if denied
        self._validate_target()         # raises ValueError on bad URL/SSRF
        self.start_time = time.time()
        await self._phase_crawl()
        await self._phase_test()
        self._deduplicate_vulnerabilities()
        self._emit_summary()
        return self.vulnerabilities

    async def _phase_crawl(self):
        self.logger.info("[PHASE 1] Crawling")
        async with AsyncWebCrawler(self.config, self.logger) as crawler:
            self.pages = await crawler.crawl()
            self._page_cache = crawler.response_cache

    async def _phase_test(self):
        self.logger.info(f"[PHASE 2] Testing {len(self.pages)} pages")
        tester = AsyncVulnerabilityTester(self.config, self.logger)
        coros = [tester.test_page(p, self._page_cache.get(p))
                 for p in self.pages if self._scan_active]
        results = await asyncio.gather(*coros, return_exceptions=True)
        for r in results:
            if isinstance(r, list): self.vulnerabilities.extend(r)

    def _emit_summary(self):
        duration = time.time() - self.start_time
        self.logger.info(f"Scan complete in {duration:.1f}s — "
                         f"{len(self.vulnerabilities)} findings")

import asyncio, os

celery = Celery("sentinal",

@celery.task(bind=True, max_retries=0)
def run_scan_task(self, scan_id: str, config_dict: dict):
    loop = asyncio.new_event_loop()
    try:
        config = ScanConfig(**config_dict)
        scanner = VulnerabilityScanner(
            config,
            logger=RedisPublishLogger(scan_id)
        )
        vulns = loop.run_until_complete(scanner.scan())
        loop.run_until_complete(store.save_results(scan_id, vulns))
        loop.run_until_complete(store.set_status(scan_id, "complete"))
    except Exception as e:
        loop.run_until_complete(store.set_status(scan_id, "failed", str(e)))
    finally:
        loop.close()

The current crawler parses static HTML only. React, Vue, and Angular SPAs — which now represent the majority of modern web apps — are effectively invisible. # backend/playwright_crawler.py
from playwright.async_api import async_playwright

class PlaywrightCrawler(AsyncWebCrawler):
    async def crawl_page(self, url: str):
        async with async_playwright() as pw:
            browser = await pw.chromium.launch(headless=True)
            ctx = await browser.new_context(
                user_agent=self.config.user_agent,
                ignore_https_errors=not self.config.verify_ssl
            )
            page = await ctx.new_page()
            # Block images/fonts for speed
            await page.route("**/*.{png,jpg,gif,woff2,svg,mp4}",
                             lambda r: r.abort())
            await page.goto(url,
                wait_until="networkidle",
                timeout=self.config.timeout * 1000)
            html = await page.content()
            links = self._extract_links(url, html)
            await browser.close()
        return url, links, html, True

# ScanConfig — new field
use_browser: bool = False

# Crawler factory
def make_crawler(config, logger):
    return PlaywrightCrawler(config, logger) if config.use_browser \
           else AsyncWebCrawler(config, logger)

# backend/ai_triage.py
import anthropic, json, asyncio

_client = anthropic.AsyncAnthropic()

TRIAGE_PROMPT = """You are a senior security engineer.
Given this finding, respond with ONLY a JSON object:
{{
  "adjusted_severity": "Critical"|"High"|"Medium"|"Low",
  "exploitability": "one sentence, max 20 words",
  "code_fix": "minimal code snippet, detected language from URL",
  "confidence_note": "why severity was adjusted or kept"
}}

Finding:
Type: {vuln_type}
URL: {url}
Description: {description}
Evidence: {evidence}
Current severity: {severity_level}"""

async def triage_finding(vuln: Vulnerability) -> dict:
    msg = await _client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=500,
        messages=[{"role":"user", "content":
            TRIAGE_PROMPT.format(**vuln.to_dict())}]
    )
    return json.loads(msg.content[0].text)

async def triage_all(vulns: list[Vulnerability],
                      concurrency: int = 5) -> list[dict]:
    sem = asyncio.Semaphore(concurrency)
    async def _guarded(v):
        async with sem: return await triage_finding(v)
    return await asyncio.gather(*[_guarded(v) for v in vulns],
                                return_exceptions=True)

PostgreSQL scan history + diff engine
Detect new vulnerabilities across scans — the key CI/CD integration feature
HIGH Postgres
10 hrs high
What & why
Teams care far more about regressions than a stable list of known issues. The diff engine compares two scans for the same target by fingerprinting each finding as md5(vuln_type || url || description). Only fingerprints present in the new scan but absent from the baseline are surfaced as new findings. This is what makes the GitHub Actions integration actually useful.
Implementation
-- migrations/001_create_scans.sql
CREATE TABLE scans (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    target_url TEXT NOT NULL,
    started_at TIMESTAMPTZ DEFAULT now(),
    completed_at TIMESTAMPTZ,
    status TEXT NOT NULL DEFAULT 'queued',
    config JSONB,
    user_id UUID    -- populated in Phase 4
);

CREATE TABLE findings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    scan_id UUID REFERENCES scans(id) ON DELETE CASCADE,
    vuln_type TEXT NOT NULL,
    url TEXT NOT NULL,
    description TEXT,
    severity_level TEXT NOT NULL,
    severity_score FLOAT,
    evidence TEXT,
    remediation TEXT,
    confidence TEXT,
    fingerprint TEXT GENERATED ALWAYS AS (
        md5(vuln_type || '|' || url || '|' || description)
    ) STORED
);

CREATE INDEX idx_findings_fp ON findings(fingerprint);
CREATE INDEX idx_scans_target ON scans(target_url, started_at DESC);

-- Diff query: new findings in scan B not in scan A
SELECT f2.*
FROM findings f2
WHERE f2.scan_id = :new_scan_id
  AND f2.fingerprint NOT IN (
    SELECT fingerprint FROM findings
    WHERE scan_id = :baseline_scan_id
  )
ORDER BY
  CASE f2.severity_level
    WHEN 'Critical' THEN 1 WHEN 'High' THEN 2
    WHEN 'Medium' THEN 3 ELSE 4 END;

from selectolax.parser import HTMLParser

def _extract_links(self, url: str, html: str) -> set[str]:
    """5-10x faster than BeautifulSoup for link extraction."""
    tree = HTMLParser(html)
    links = set()
    for node in tree.css("a[href], form[action]"):
        href = (node.attributes.get("href") or
                node.attributes.get("action", ""))
        if href and self._is_valid_link(url, href):
            full = urljoin(url, href).split("#")[0]
            links.add(full)
    return links

# Keep bs4 ONLY for complex form field traversal
def _extract_forms(self, url: str, html: str):
    from bs4 import BeautifulSoup
    soup = BeautifulSoup(html, "html.parser")
    return soup.find_all("form")

@app.get("/scan/{scan_id}/stream")
async def stream_results(scan_id: str):
    async def generator():
        last_len = 0
        while True:
            results = await store.get_results(scan_id)
            for r in results[last_len:]:
                yield f"event: finding\ndata: {json.dumps(r)}\n\n"
            last_len = len(results)
            status = await store.get_status(scan_id)
            if status in ("complete", "failed"):
                yield f"event: done\ndata: {status}\n\n"
                break
            await asyncio.sleep(0.5)

    return StreamingResponse(generator(),
        media_type="text/event-stream",
        headers={"Cache-Control":"no-cache",
                 "X-Accel-Buffering":"no"})

        team_id: str,

REPORT_SYSTEM = """You are a security engineer writing a
penetration test report for a development team.
Be direct, technical, and constructive. One sentence per idea."""

async def generate_finding_section(
        vuln: Vulnerability, triage: dict) -> str:
    lang = _detect_language(vuln.url)
    prompt = f"""Write a finding section. Include:
- Business impact (1 sentence)
- Step-by-step reproduction (numbered list)
- CVSS 3.1 vector and score
- Code fix in {lang}
Max 400 words. Format: Markdown.

Finding: {json.dumps(vuln.to_dict())}
AI triage: {json.dumps(triage)}"""
    msg = await _client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=600, system=REPORT_SYSTEM,
        messages=[{"role":"user","content":prompt}])
    return msg.content[0].text

async def generate_pdf_report(scan_id: str) -> bytes:
    results = await store.get_results(scan_id)
    vulns = [Vulnerability(**r) for r in results]
    triages = await triage_all(vulns)
    sections = await asyncio.gather(*[
        generate_finding_section(v, t)
        for v, t in zip(vulns, triages)])
    jinja_env = Environment(loader=PackageLoader("backend"))
    html_str = jinja_env.get_template("report.html.j2").render(
        scan_id=scan_id, sections=sections, findings=results)
    return HTML(string=html_str).write_pdf()

// extension/src/extension.ts
import * as vscode from 'vscode';

export function activate(ctx: vscode.ExtensionContext) {
    const diag = vscode.languages.createDiagnosticCollection('sentinalscan');
    ctx.subscriptions.push(diag);

    ctx.subscriptions.push(
        vscode.commands.registerCommand(
            'sentinalscan.loadFindings', async () => {
                const cfg = vscode.workspace.getConfiguration('sentinalscan');
                const apiBase = cfg.get<string>('apiBase');
                const scanId = cfg.get<string>('lastScanId');
                const res = await fetch(`${apiBase}/scan/${scanId}/results`);
                const findings = await res.json();
                _mapFindingsToEditor(findings, diag);
            }
        )
    );
}

function _mapFindingsToEditor(findings, diag) {
    const byFile = new Map();
    for (const f of findings) {
        const uri = _urlToFileUri(f.url);
        if (!byFile.has(uri)) byFile.set(uri, []);
        byFile.get(uri).push(new vscode.Diagnostic(
            new vscode.Range(0, 0, 0, 0),
            `[SentinalScan] ${f.vuln_type}: ${f.description}`,
            _severityToVSCode(f.severity_level)
        ));
    }
    diag.clear();
    byFile.forEach((d, u) => diag.set(vscode.Uri.parse(u), d));
}

Tech stack changes

HTTP client
requests→httpx
Identical API. Adds async, HTTP/2, connection limits. Required prerequisite for the entire async migration. Drop-in for 95% of usage.
HTML parsing
bs4 (links)→selectolax
C-backed parser, 5–10× faster for CSS selector queries. Keep bs4 only for form field traversal where its navigation API is irreplaceable.
Task queue
threading.Thread→Celery + Redis
Decouples HTTP from compute. Horizontal worker scaling, built-in retry, task result TTL, and rate limiting per queue.
State & persistence
dict (in-process)→Redis + Postgres
Redis for ephemeral state and pub/sub. Postgres for permanent scan history, user accounts, and finding diffs. Survives restarts.
Authentication
static API key→fastapi-users JWT
Per-user token rotation, email verification, password reset — all included. Async-native, Pydantic v2 compatible, pluggable OAuth.
Browser automation
(none)→Playwright
Covers SPAs and JS-rendered apps invisible to the static crawler. Async API matches the rest of the stack. Network interception for payload injection.
Logging
stdlib logging→structlog
JSON output with bound context (scan_id, target). Drop into ELK, Datadog, or CloudWatch with zero config changes.
PDF generation
HTML string concat→Jinja2 + WeasyPrint
Separates template from logic. WeasyPrint renders CSS-styled HTML to PDF with page numbers and table-of-contents via CSS Paged Media spec.
Frontend query
TanStack Query→keep
Already the right choice. Fix the dead options key in useScanResults and gate enabled on a non-null previous scan ID. No library change needed.
Data validation
Pydantic (partial)→Pydantic v2 full
Add @field_validator on target_url (URL format), max_pages (1–500 range), workers (1–20). Errors surface as 422s before the scan starts.

Portfolio README structure

The 90-second rule
A hiring committee will spend 90 seconds on the README before deciding whether to look deeper. The structure below is optimized for that window.
Paragraph 1 — What it does and scale
What it does, who it's for, one sentence on measurable scale: "Scans 200+ pages/minute with async concurrency, detecting 12 vulnerability classes including SQLi, XSS, and SSRF."
Paragraph 2 — Architecture decision rationale
Why httpx over requests, why Celery over threading, why Redis pub/sub for real-time logs. This signals choices were deliberate and demonstrates systems thinking — the highest-signal section for a senior role.
Section: Security hardening decisions
A bullet list of the specific vulnerabilities you found in the original design and fixed. For a security engineering role this is the most important section — you attacked your own tool.
Section: Before/after performance numbers
The benchmark table: crawl time from 90s to under 20s, API response from blocking to under 50ms. Concrete numbers matter more than vague performance claims.
One-command setup — non-negotiable
docker compose up must work on a fresh machine. If a reviewer cannot get it running in 3 minutes they close the tab. This is the single most important non-code element of the portfolio.
Architecture diagram in the repo
Link directly to an SVG committed to the repo, not an external service. GitHub renders SVGs inline in Markdown. The target architecture diagram from this roadmap is ready to commit.
